# Contributing

本项目是长期企业级项目，所有贡献必须服务于工业装备渠道增长平台的长期目标。

## 核心原则

1. 商业目标优先：加盟合作、终端询盘、品牌建设。
2. 架构边界优先：不为短期速度破坏长期维护性。
3. 中国 SEO 优先：面向百度、360、搜狗、神马、微信搜一搜、知乎、抖音、小红书。
4. GEO 优先：面向 DeepSeek、豆包、腾讯元宝、Kimi、通义千问。
5. 合规优先：不得编造收益、案例、参数、行业排名或授权范围。

## 开发流程

1. 阅读 `README.md`、`PROJECT_PRD.md`、`ROADMAP.md`、`docs/MEMORY.md`。
2. 明确变更属于 Website、SEO、GEO、CRM、Partner、Marketing、Dashboard 或 Knowledge Base。
3. 创建 Issue 或关联已有 Issue。
4. 提交 Pull Request。
5. 通过 GitHub Actions。
6. 更新 `CHANGELOG.md`、`TODO.md`、`ROADMAP.md` 和相关 docs 文档。

## Commit 规范

提交信息必须语义化，推荐格式：

```text
[M0] Repository Bootstrap
[M1] Website Foundation
[M2] SEO Foundation
[M3] GEO Knowledge Base
[M4] CRM Lead System
```

禁止使用无语义提交信息，例如 `update`、`fix`、`修改`、`保存`。

## Pull Request 要求

PR 必须说明：

- 变更范围
- 架构影响
- SEO/GEO 影响
- 风险与回滚方式
- 验证结果
- 文档更新情况

## 合规要求

招商合作内容禁止使用：

- 保本
- 稳赚
- 零风险
- 快速暴富
- 未经证实的行业第一
- 虚构客户案例
- 虚构收益周期

所有产品参数、客户案例、授权范围和合作政策必须来自真实资料。

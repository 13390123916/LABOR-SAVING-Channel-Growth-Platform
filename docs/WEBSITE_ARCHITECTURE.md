# Website Architecture

文档职责：定义 LABOR-SAVING 官网信息架构、导航层级和页面边界。

## 1. 架构目标

官网信息架构必须服务三个目标：

1. 招募经销商、区域代理和渠道合作伙伴。
2. 获取终端工业客户的产品咨询、工况咨询和项目线索。
3. 为中国 SEO、Google 可发现性和 AI Search 长期内容沉淀提供稳定 URL 与结构化内容基础。

## 2. 一级导航

一级导航按商业权重排序：

| 导航 | 权重 | 核心目标 |
| --- | --- | --- |
| 首页 | 基础入口 | 分流渠道合作、产品、行业和解决方案 |
| 加盟合作 | ★★★★★ | 获取经销商、代理商和项目合作伙伴线索 |
| 产品中心 | ★★★★★ | 承接产品理解、选型和渠道扩品需求 |
| 行业应用 | ★★★★ | 承接行业工况搜索和终端客户咨询 |
| 解决方案 | ★★★★ | 承接组合方案、项目型需求和高意向询盘 |
| 知识中心 | ★★★ | 承接 SEO/GEO 内容沉淀和 AI Search 引用 |
| 关于 LABOR-SAVING | 信任背书 | 承接企业实体、资质和联系方式 |


## 4. 页面职责

| 页面模块 | 核心职责 | 转化目标 |
| --- | --- | --- |
| 首页 | 建立品牌、产品线、行业场景和加盟合作入口 | 分流到加盟合作、产品、行业应用、解决方案和咨询入口 |
| 加盟合作 | 定义为 LABOR-SAVING 渠道增长中心，不是普通招商页面 | 渠道合作线索、区域合作申请、CRM 入口 |
| 产品中心 | 承载产品分类、型号和解决方案入口 | 产品咨询、选型咨询、渠道扩品 |
| 行业应用 | 按工况和行业组织内容 | 终端项目询盘、行业长尾 SEO |
| 解决方案 | 承载跨产品、跨行业的项目型方案入口 | 高意向项目咨询、方案沟通 |
| 知识中心 | 沉淀技术文章、案例、FAQ 和下载内容 | SEO/GEO 收录、AI Search 引用 |
| 关于 LABOR-SAVING | 承载企业实体、资质和联系方式 | 信任背书、联系方式确认 |

## 5. 加盟合作页面定位

加盟合作页面不得作为普通“招商页面”处理，必须定义为：

```text
LABOR-SAVING 渠道增长中心
```

推荐主 URL：

```text
/partner/
```

可选兼容路径：

```text
/distributor/
/join/
```

兼容路径只作为后续重定向或投放短链规划，不作为主要 SEO 页面。

## 6. 加盟合作页面结构权重

### 6.1 首屏价值 20%

目标：3 秒告诉代理商为什么加入。

首屏表达：

```text
成为 LABOR-SAVING 授权合作伙伴

共享工业智能搬运增长市场

气动助力机械臂
液压扭矩工具
重载装配解决方案
```

### 6.2 市场机会 20%

关键词：

- 工业自动化升级
- 重载搬运需求
- 风电装备市场
- 石油化工维修市场
- 智能制造趋势

目的：让代理商看到明确的工业增长场景。

### 6.3 合作模式 25%

重点合作类型：

```text
区域代理

行业代理

渠道经销商

项目合作伙伴
```

必须说明：

- 区域保护
- 价格体系
- 技术支持
- 营销支持
- 培训支持

注意：以上属于合作机制栏目规划，具体政策、价格体系和保护范围必须以后续真实商务资料为准。

### 6.4 八大渠道赋能 20%

重点建设：

```text
产品赋能

技术赋能

销售赋能

市场赋能

内容赋能

培训赋能

售后赋能

品牌赋能
```

该模块用于形成区别于普通厂家招商页的渠道增长中心优势。

### 6.5 留资转化 15%

转化动作不使用普通“联系我们”，而使用：

```text
申请成为区域合作伙伴
```

CRM 入口字段：

- 公司名称
- 所在地区
- 主营产品
- 客户行业
- 销售团队规模
- 已有渠道资源

## 7. 结构边界

- 本文档只定义官网信息架构，不编写页面正文。
- LS40、LS60、LS70 仅作为栏目和 URL 规划，不代表产品参数、配置和价格已经确认。
- 应用案例栏目不得编造客户名称、收益结果或未授权案例。
- 合作政策栏目不得承诺保本、收益、区域独家或未经确认的商务条件。
- 认证资质栏目只展示真实可核验资料。

## 8. M2 前置要求

进入 M2 Website Development 前，必须先确认：

- 每个一级栏目 URL 已进入 SEO/GEO 蓝图。
- 每类页面 SEO 字段标准已定义。
- 内容生产规则已进入 `docs/CONTENT_SYSTEM.md`。
- CI 已检查架构文档、SEO/GEO 蓝图和内容系统文档存在。
